# Описание структуры проекта

Документ описывает назначение основных директорий и ключевых модулей монорепозитория `family-tree`.

## Корень репозитория

- `docker-compose.yml` — поднимает полный стек: PostgreSQL, Redis, MinIO, backend, frontend, nginx.
- `README.md` — основной вход в проект: запуск, стек, ссылки.
- `scripts/` — утилиты для генерации больших тестовых деревьев.
- `docs/` — документация проекта (операторская, испытания, пояснительная записка).
- `docs_2025/` — архив/дополнительный пакет документов.

## Backend (`backend/`)

Java/Spring Boot приложение с REST API.

### Основные директории

- `src/main/java/com/example/backend/` — исходный код.
- `src/main/resources/` — конфигурация (`application.yml`), ресурсы.
- `src/test/java/` — место для тестов.
- `Dockerfile` — образ backend-сервиса.
- `pom.xml` — зависимости и сборка Maven.

### Паттерн организации модулей

Большинство доменных областей построены по слою:

- `api/` — контроллеры (`@RestController`).
- `application/` — сервисы с бизнес-логикой.
- `dto/` — входные/выходные модели API.
- `entity/` — JPA-сущности.
- `repository/` — Spring Data репозитории.

### Ключевые доменные модули

- `auth/` — регистрация, логин, профиль.
- `person/` — управление персонами дерева.
- `relationship/` — родственные/брачные связи.
- `event/` — события и привязка к персонам.
- `tree/` — деревья, участники, роли, ветки.
- `tree/invite/` — приглашения и токены.
- `graph/` — подготовка данных для граф-визуализации.
- `media/` — метаданные файлов и работа с медиа.
- `gedcom/` — импорт/экспорт GEDCOM/JSON.
- `version/` — рабочие копии, снапшоты, слияния.
- `audit/` — аудит действий.
- `shared/` — общие сущности, исключения, storage/email-конфиг.
- `security/` и `config/` — JWT, фильтры, конфигурация безопасности.

## Frontend (`frontend/`)

React SPA (Vite), пользовательский интерфейс проекта.

### Основные директории

- `src/pages/` — экранные страницы и роуты.
- `src/components/` — переиспользуемые компоненты.
- `src/components/tree/` — визуализация и инструменты дерева.
- `src/components/person/` — карточка персоны, медиа, события, связи.
- `src/components/members/` — участники и права.
- `src/components/version/` — UI для версий/слияний.
- `src/components/ui/` — базовые UI-компоненты (модалки, спиннеры, тосты).
- `src/api/` — клиентские модули работы с backend API.
- `src/context/` — глобальные контексты (авторизация, тосты).
- `src/utils/` — утилиты и алгоритмы (layout, статистика, экспорт).
- `src/index.css` — основной стиль приложения.

### Точки входа и конфиги

- `src/main.jsx` — инициализация React приложения.
- `src/App.jsx` — маршрутизация и общая композиция.
- `vite.config.ts` — конфиг Vite.
- `eslint.config.js` — линтинг.
- `Dockerfile` — образ frontend-сервиса.

## Nginx (`nginx/`)

- `nginx.conf` — reverse proxy:
  - `/` -> frontend
  - `/api/` -> backend
  - `/media/` -> MinIO bucket
- `Dockerfile` — образ nginx-шлюза.

## Поток запросов

1. Пользователь обращается к `http://localhost`.
2. Nginx отдаёт frontend.
3. Запросы к `/api/*` проксируются в backend.
4. Медиа по `/media/*` проксируются в MinIO.

## Хранилища и инфраструктура

- PostgreSQL — основная реляционная база.
- Redis — кеш и служебные данные.
- MinIO — объектное хранилище медиа.

## Рекомендация по запуску

Для согласованного окружения используйте запуск через Docker Compose из корня репозитория:

```bash
docker compose up --build -d
```
